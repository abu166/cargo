# Payment Service

Микросервис управления платежами и подтверждением платежей.

## Функционал

- Создание платежей
- Получение информации о платежах
- Подтверждение платежей
- Синхронизация со статусом отправлений
- Логирование аудита платежей

## Статусы платежей

1. PENDING - Ожидание подтверждения
2. CONFIRMED - Подтвержден
3. FAILED - Не удался
4. REFUNDED - Возвращен

## API Endpoints

- `POST /payments` - Создание платежа
- `GET /payments/:id` - Получение платежа
- `POST /payments/:id/confirm` - Подтверждение платежа

## Интеграция

Этот микросервис взаимодействует с:
- Shipment Service - для обновления статуса отправлений
- Auth Service - для аутентификации пользователей
